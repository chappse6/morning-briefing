◐ Idea of the Day · 05 / 16

Agent Auth Layer — OAuth still treats an agent like a person

AI 에이전트가 사용자 대신 외부 서비스에 접속할 때, 현행 OAuth/OIDC는 "사람"을 전제로 설계되어 있어 per-agent identity · capability-based authorization · 에이전트 라이프사이클이 빠져있습니다. 세 개의 에이전트가 같은 MCP 서버에 접속해도 서버는 한 명의 클라이언트로 인식. 에이전트 전용 인증 레이어(Agent Auth · WIMSE · SPIFFE 기반 AIMS)가 떠오르는 이유.
